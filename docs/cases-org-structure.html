App routes (pages)

  • /crm/cases Cases list: list view, kanban, and tab modes. Sidebar list + main content (detail or empty). New Case modal, filters (status, priority, SLA). page
  • /crm/cases/[id] Case detail: full case view with AccountContextPanel, CaseDetailContent, Link Case modal, notes/call/email panels. page
  • /crm/cases/summary Cases summary: charts and analytics (trends, created/closed, status distribution, etc.). page

API routes

  • GET /api/cases List all cases, or ?caseNumber=X for one by number. API
  • POST /api/cases Create a case. API
  • GET /api/cases/[id] Get one case by id. API
  • PATCH /api/cases/[id] Update a case. API
  • DELETE /api/cases/[id] Delete a case. API
  • POST /api/cases/[id]/attachments Upload one or more files for a case (FormData, field name "files"). API
  • GET /api/cases/[id]/attachments/[attachmentId] Download or view attachment; ?inline=true for inline (e.g. PDF preview). API

Types (crm.ts)

  • CaseItem id, caseNumber, accountId, accountName, type, subType, status, priority, slaStatus, slaDeadline, owner, team, description, resolution, communications, activities, attachments, relatedCases, parentCaseId, childCaseIds, pendingReason, descriptionHistory, …
  • CaseType "Complaint" | "Enquiry" | "EWR" | "Onboarding" | "Dunning"
  • CaseStatus "New" | "In Progress" | "Pending" | "Resolved" | "Closed"
  • CasePriority "Critical" | "High" | "Medium" | "Low"
  • SLAStatus "On Track" | "At Risk" | "Breached"

Data flow

  • Mock When NEXT_PUBLIC_USE_DATABASE ≠ "true", cases come from mock data + session (session-cases). getCaseById, getSessionCase, mergeMockWithSession.
  • Database When DB enabled, list/detail CRUD and attachments go through /api/cases/*; Prisma + lib/db (prismaCaseToCaseItem, caseItemToPrismaPayload).
  • Case numbers Generated via generateCaseNumber(accountName, sequence) from lib/case-number (e.g. BBMC-2026-001847).
